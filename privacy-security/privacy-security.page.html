Privacy & Security Mobile App Privacy & Security Requirements

Data Protection

All communication between the app and the remote REST API uses HTTPS, ensuring encrypted data transmission. No sensitive personal information is stored locally on the device.

Authentication & Authorization

The current version does not require user login. However, future iterations should implement role-based access control to prevent unauthorized modifications.

Input Validation

All input fields are validated on the client side (type checking, required fields, numeric ranges) to prevent malformed data submission. Server-side validation is also enforced by the API.

API Security

The REST API uses standard HTTP methods. Delete operations are restricted for certain items (e.g., "Laptop") to maintain at least one record in the database.

Data Integrity

Inventory item names are unique identifiers. The system prevents duplicate entries and enforces data types consistent with the database schema.

User Privacy

No personal data is collected from users. Supplier names and item details are only used for inventory management purposes and are not shared with third parties.

Secure Development Practices

The application is built with Ionic and Angular, following security best practices including proper error handling, avoiding inline scripts, and using Angular's built-in sanitization for data binding.
